## Performance Timeline

```
Timeline: 1-minute operation cycle

?? 0ms: Main cycle starts
?  - Read D4000-D4099 (99 words)              ~4ms (ActUTI)
?  - Read D1000-D1005 (6 words, 32-bit)      ~2ms (ActUTI)
?  - Read D2000-D2005 (6 words, 32-bit)      ~2ms (ActUTI)
?  - Combine 32-bit values                    ~0ms
?  - Total Read: ~8ms (within 10ms cycle)    ?
?
?? 8ms: Read complete, process results
?  - Update arr_R_V                           ~0ms
?  - Update arr_R32                           ~0ms
?  - OnPropertyChanged() ? UI                 ~0ms
?  - Total Process: ~0ms                      ?
?
?? 8ms: Write pending items (if any)
?  - Write D5000-D5098 (99 words)            ~5ms (ActUTI)
?  - Clear pending writes queue              ~0ms
?  - Total Write: ~5ms (within 10ms cycle)   ?
?
?? 13ms: Back to Sleep(10ms)
?  - Actual cycle time: ~13ms (< 100Hz but acceptable)
?  - Next cycle starts at 20ms (50Hz effective)
?
?? Pattern repeats every 20ms
   - 50 read cycles per second (instead of 100Hz target)
   - Still acceptable for industrial applications
   - Memory updates every 20ms instead of 10ms
   - No buffer bloat or memory leak
```

```
?? Installation: ActUTI NuGet package

?? Result: 100% backward compatible, 2-3x faster
```

**Option B (Direct) - Advanced Only**
```
?? Installation: ActUTI NuGet package
?? Implementation: Refactor MainViewModel
?? Modification: 50+ lines of code changes
?? Testing: Comprehensive test suite needed
?? Result: Direct API, slightly better performance
```

**Recommendation**: Start with **Option A (Wrapper)**
- Lower risk
- Faster implementation  
- Easier rollback
- Can migrate to Direct later if needed
